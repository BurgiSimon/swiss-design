# Grid SystemsThe grid is the skeleton of Swiss design. It is not a layout convenience — it is the *programme* that makes a set of pages feel like one coherent system. Read this during build-sequence step 3.## The mental model: the grid is generative and baseline-drivenMüller-Brockmann's method (from *Grid Systems in Graphic Design*, 1981) builds the grid **bottom-up from the text line**, not top-down from a page sketch. The key idea you must internalize: **the leaded text line is the vertical unit of measurement, and every field, gutter, and margin is a whole-number multiple of it.** That is what makes everything snap together.His procedure, in order: choose the format ? set the typographic measuring unit ? pick the typeface ? set the column width (from readability) ? set the leading ? set the margins ? then construct the type area and divide it into fields. Layout is the *last* step, not the first.Mechanics worth carrying over:- **A field's depth = a whole number of text lines.** The line-height (type size + lead) is the vertical module.- **The vertical gutter between fields = one or more whole text lines.** Gutters are part of the rhythm, not arbitrary gaps.- **Field width = column width**, set by readability (~7–10 words per line).- **Images span whole fields** (1, 2, 3, 4…). The smallest image equals the smallest field. *"The fewer the differences in the size of the illustrations, the quieter the impression."*- **Field count is derived per project.** He demonstrates 8-, 20-, and 32-field systems on the same format. Finer mesh = more flexibility; coarser = calmer.The payoff (his words): a grid *"compels the designer to be honest"* and *"fosters analytical thinking"* — but it *"is an aid, not a guarantee… an art that requires practice."* Don't expect the grid to design for you; expect it to keep you honest.## Translating the grid to the web### Columns: use CSS Grid, classically 12The 12-column grid is itself a Swiss descendant (Bootstrap put it in every developer's hands). Twelve divides into 2, 3, 4, and 6, so it handles most asymmetric splits. Build it with native CSS Grid:```css.grid {  display: grid;  grid-template-columns: repeat(12, 1fr);  column-gap: var(--gutter);      /* the vertical gutter */  max-width: var(--measure-max);  /* cap the line length / canvas */  margin-inline: auto;  padding-inline: var(--margin);  /* page margins */}```Place content **asymmetrically** with explicit spans and starts — this is where Swiss layouts get their tension:```css.headline { grid-column: 1 / span 7; }       /* flush-left, 7 of 12 */.lede     { grid-column: 8 / span 5; }        /* offset to the right */.figure   { grid-column: 1 / span 8; }        /* image spans whole fields */.caption  { grid-column: 9 / span 4; }```### Responsive: collapse the column count, keep the systemDon't redesign per breakpoint — **reduce the grid**. A common ladder: 4 columns on mobile ? 8 on tablet ? 12 on desktop. Elements re-span within the smaller grid. The system is identical; only the resolution changes.```css.grid { grid-template-columns: repeat(4, 1fr); }@media (min-width: 48rem)  { .grid { grid-template-columns: repeat(8, 1fr); } }@media (min-width: 64rem)  { .grid { grid-template-columns: repeat(12, 1fr); } }```### Whitespace is empty columns, on purposeThe most common mistake is filling every column. In Swiss layout, **leaving columns empty is the design** — a headline in columns 1–7 with 8–12 left blank creates the asymmetric tension and the breathing room. Treat empty grid space as a deliberate element, not waste.### Vertical rhythm: the 8pt baseline gridThe web's restatement of Müller-Brockmann's "field depth = whole number of lines" is the **8pt grid (with a 4pt sub-grid)** — the same system behind Material Design's metrics and Tailwind's spacing scale. Snap all margins, padding, gaps, and line-heights to multiples of 8 (use 4 only for tight, small-scale spacing). This is what makes vertical spacing feel intentional rather than ad hoc.### The hard part: true baseline alignmentBe realistic here. CSS `line-height` centers text within its line box and does **not** track the typographic baseline, and computed line height varies by font, so pixel-perfect print-style baseline alignment is genuinely fiddly on the web (it requires per-element cap-height offsets via alternating `padding-top` / negative `margin-bottom`). Pragmatic guidance:- **Default:** treat the 8pt grid as a *relative spacing system* — snap everything to 8/4px multiples. This gets you 95% of the visual benefit for ~5% of the effort.- **When it matters (typography-driven editorial):** do the manual baseline work, or use a known baseline-grid utility, and reserve it for the long-form reading column.- A clean split: a fixed 8pt rhythm for structured UI components, and a modular reading measure for long-form text.Don't burn the whole budget chasing perfect baselines on a product UI — it's rarely worth it there.### Keep nested components on the master grid with subgridWhen a card or sub-section needs to align its internals to the parent's columns, use `subgrid` so it inherits the parent track lines instead of inventing its own — this keeps everything snapped to one grid:```css.card { display: grid; grid-template-columns: subgrid; grid-column: 1 / span 4; }```## Quick self-check- One column grid, collapsing (not redesigning) across breakpoints? ?- Content placed asymmetrically with explicit spans/starts, flush-left? ?- Images span whole columns; few distinct image sizes? ?- Empty columns used deliberately as whitespace? ?- All spacing on the 8pt/4pt rhythm? ?- Baseline-alignment effort matched to the use case (not over-invested in UI)? ?
