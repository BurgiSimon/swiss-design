# Implementation: The System in Any StackThe Swiss system is identical across technologies — only the syntax for expressing tokens and grids changes. This file gives a tech-agnostic CSS core, then short notes for the common stacks. Read it during build-sequence step 2 once you've chosen a stack. Don't tie the design to a framework; tie it to the tokens.## The tech-agnostic core (plain CSS custom properties)This works everywhere and is the reference implementation. Every other stack is a re-expression of this.```css:root {  /* spacing — 8pt system, 4px base */  --space-1: 4px;  --space-2: 8px;  --space-3: 12px; --space-4: 16px;  --space-6: 24px; --space-8: 32px; Define tokens in `@theme` so they become utilities:```css@import "tailwindcss";@theme {  --font-sans: "Inter", -apple-system, BlinkMacSystemFont, "Helvetica Neue", Arial, sans-serif;  --text-caption: 0.8rem; --text-h4: 1.25rem; --text-h3: 1.5625rem;  --text-h2: 1.953rem;    --text-h1: 2.441rem; --text-display: 3.05rem;  --color-ink: #111111;   --color-paper: #ffffff;  --color-gray-1: #6b6b6b; --color-gray-2: #d4d4d4; --color-accent: #e2231a;  --leading-tight: 1.1; --leading-snug: 1.25; --leading-body: 1.5;}```Usage: `grid grid-cols-4 md:grid-cols-12 gap-6`, `col-span-7`, `col-start-8`, `text-h1 font-medium leading-tight tracking-tight`, `text-ink bg-paper`, `text-accent`, `max-w-[60ch]`. Restrict yourself to the 8pt rungs (`gap-2/4/6/8/12/16`, etc.) — the discipline is the point. (Tailwind v3: put the same values under `theme.extend` in `tailwind.config.js`.)A representative section:```jsx<section className="grid grid-cols-4 md:grid-cols-12 gap-4 md:gap-6 max-w-screen-xl mx-auto px-4 md:px-8 py-24">  <h1 className="col-span-4 md:col-span-7 text-h1 font-bold leading-tight tracking-tight text-ink">    Form follows function  </h1>  <p className="col-span-4 md:col-span-5 md:col-start-8 leading-body max-w-[60ch] text-ink">    Body set flush-left, ragged-right, on a constrained measure.  </p>  <figure className="col-span-4 md:col-span-8 mt-12">{/* image spans whole fields */}</figure>  <aside className="col-span-4 md:col-span-4 text-caption text-gray-1 mt-12">{/* caption column */}</aside></section>```## React / CSS-in-JSExpress tokens as a JS theme object and consume via your styling solution (styled-components, vanilla-extract, CSS Modules, inline style with CSS vars). The structure mirrors the core above:```jsexport const tokens = {  space: { 1:4, 2:8, 3:12, 4:16, 6:24, 8:32, 12:48, 16:64, 24:96, 32:128 },  text:  { caption:'0.8rem', base:'1rem', h4:'1.25rem', h3:'1.5625rem', h2:'1.953rem', h1:'2.441rem', display:'3.05rem' },  leading: { tight:1.1, snug:1.25, body:1.5 },  color: { ink:'#111', paper:'#fff', gray1:'#6b6b6b', gray2:'#d4d4d4', accent:'#e2231a' },  grid: { gutter:'1.5rem', margin:'2rem', measure:'60ch', canvasMax:'72rem' },};```Build a small set of primitives (`<Grid>`, `<Col span start>`, `<Text variant>`) so screens are composed from the system, not from ad-hoc styles — this is the component-level expression of "programme over decision." Avoid HTML `<form>` tags inside artifacts; wire interactions with event handlers.## Vue / Svelte / otherSame tokens, exposed as CSS custom properties in a global stylesheet (the core block above) and consumed in scoped styles, or via the framework's theming. The grid is plain CSS Grid in all of them. Nothing about the system is framework-specific — keep the tokens in one place and reference them everywhere.## Plain HTML/CSSUse the core block verbatim. This is also the most performant Swiss path: prefer the system-font stack (zero webfont cost), one stylesheet, no framework. Swiss design's restraint makes it naturally fast.## Performance & accessibility notes (all stacks)- **Fonts:** system stack = no network cost. If loading Inter, subset it, use `font-display: swap`, and prefer the variable font to avoid multiple weight files.- **Contrast:** verify every token pair against WCAG AA before shipping (see `color-and-tokens.md`). This is the highest-leverage QA step.- **Semantics:** real heading levels (`h1`–`h6`) in scale order, real landmarks, focus-visible states (the accent color is a natural focus ring). High contrast + left alignment + size hierarchy means Swiss is accessible by default — don't undo it with low-contrast "elegance."- **Motion:** if any, keep it minimal and functional (a single staggered reveal, calm transitions); respect `prefers-reduced-motion`. Ornamental motion is off-style.
