# Redesign Playbook + Anti-Patterns + QA ChecklistTwo jobs live here: (1) the method for converting an *existing* design into Swiss style (workflow B), and (2) the anti-pattern catalogue and final QA checklist that **both** workflows run before shipping. Read the relevant part during the redesign workflow and always during build-sequence step 5.## Converting an existing design to Swiss styleThe mistake to avoid is "repainting the surface" Ś swapping in Helvetica and a red button while leaving the old structure intact. Swiss redesign means **replacing the system, not the skin**. Work in this order:### 1. Audit the current design against the systemCatalogue what's actually there, mapped to the five Swiss levers:- **Grid:** is there a consistent column grid, or is everything ad-hoc? Note the real structure hiding under the current layout.- **Type:** how many families, weights, sizes? (Usually far too many.) What's the body size, line-height, measure, alignment?- **Color:** count the colors. (Usually far too many.) Which carry meaning vs decorate?- **Spacing:** is spacing on any system, or arbitrary? - **Hierarchy:** is it built from size/weight/space, or from color/boxes/ornament?- **Content:** what is essential vs decorative? (You'll *keep* essential content Ś reduction is not deletion.)### 2. Define the target system (tokens first)Before touching layout, build the token set per `color-and-tokens.md`, choosing the decision-framework row that matches the product. This is the new "programme." Everything downstream is derived from it.### 3. Rebuild structure on the gridRe-lay the existing content onto a real column grid (`grid-systems.md`): asymmetric, flush-left, images on whole columns, deliberate empty columns. Keep the content; change the container.### 4. Reduce to the systemNow collapse the excess: many fonts ? one neo-grotesque; many weights ? two; many colors ? ink/paper/grays/one accent; arbitrary spacing ? the 8pt scale; decorative hierarchy ? size/weight/space. **Reduction rule:** remove ornament freely; never remove something users need to accomplish their task. If you're unsure whether an element is ornament or function, it's function Ś keep it.### 5. Re-introduce energyA reduced layout can feel dead. Bring it to life the Swiss way: strong scale contrast in the type, one decisive accent placement, and strong objective photography or a bold graphic block on the grid Ś not by adding back decoration.### 6. Run the QA checklist below.A good way to present a redesign: show the before/after mapping per lever ("7 fonts ? 1; 12 colors ? 4; no grid ? 12-col; arbitrary spacing ? 8pt") so the user sees that the *system* changed, not just the paint.## Anti-patterns (the catalogue Ś watch for these in both workflows)Each is a common way Swiss design goes wrong, with the fix.1. **Austerity masquerading as minimalism.** Stripping things users need and calling it clean. *Fix:* reduction means "only what is necessary," not "as little as possible." If removing it hurts the task, keep it.2. **Low-contrast "elegant" type.** Light-gray body text, tiny sizes, hidden nav. This violates Swiss clarity *and* fails accessibility Ś and most of the web already fails contrast, so it's the likeliest defect. *Fix:* near-black body, AA-compliant grays, real sizes. Legibility is non-negotiable.3. **Too many fonts / weights / sizes.** Trying to create interest through variety. *Fix:* one family, two weights, a small modular scale. Interest comes from contrast within restraint.4. **Abandoning the grid when it's inconvenient.** One-off placements that ignore the columns. *Fix:* the grid governs every element at every breakpoint; if it fights you, re-span within the grid rather than escaping it.5. **Fear of whitespace / cramming.** Filling every column and gap. *Fix:* whitespace is active structure Ś leave columns empty, use the big spacing steps between sections.6. **Sterile, lifeless result.** Technically minimal but dead. *Fix:* scale contrast, one bold accent, strong objective imagery.7. **Literal 1960s-print translation.** Metal-era Helvetica at 12px, fixed print proportions on a fluid screen. *Fix:* screen-optimized faces (Inter / Helvetica Now / SF), responsive grid, interaction and motion considered for screens.8. **Homogeneity / sameness.** Looks like every other grid-plus-Helvetica site. *Fix:* differentiate through distinctive imagery, editorial voice, and a considered accent Ś not more type variety.9. **Decorative color.** Using the palette to "brighten." *Fix:* color signals hierarchy/action only; expand with tints of one hue.10. **Forcing Swiss on the wrong brand.** Applying cold neutrality to a warmth-, heritage-, or play-driven brand. *Fix:* recognize the mismatch; blend in humanist type / a softer palette, or recommend a different idiom, and tell the user why.## Final QA checklist (run before shipping Ś new builds and redesigns)**System**- [ ] Tokens (grid, type scale, spacing, palette) defined and actually used everywhere Ś no off-system one-offs.- [ ] One neo-grotesque family; =2 weights.- [ ] All spacing on the 8/4 scale; generous section whitespace present.**Grid & layout**- [ ] One column grid, collapsing across breakpoints (not redesigned per breakpoint).- [ ] Content asymmetric, flush-left; images span whole columns; empty columns used deliberately.**Type**- [ ] Modular scale with a ratio matched to the use case; hierarchy from size/weight/space only.- [ ] Body 16¢18px, line-height ~1.5, measure ~60ch, flush-left/ragged-right.**Color & contrast**- [ ] Palette = ink + paper + two grays + one accent; accent used sparingly for hierarchy/action.- [ ] **Every text/background pair passes WCAG AA** (recheck gray-on-paper and accent-on-paper). Highest-leverage check.**Reduction & life**- [ ] Nothing users need was removed for "cleanliness."- [ ] The design has energy (scale contrast + accent + imagery), not just emptiness.**Fit**- [ ] Swiss neutrality actually suits this brand/use case; if not, the deviation is intentional and explained.**Accessibility & performance**- [ ] Real heading order, landmarks, visible focus states; `prefers-reduced-motion` respected.- [ ] Fonts performant (system stack or subset/variable Inter with `font-display: swap`).
